# CardCheck QA

## 1. 프로젝트 소개

**CardCheck QA**는 학술형 인스타그램 카드뉴스의 오류를 검증하는 AI 챗봇 프롬프트 패키지이다.
사용자가 카드뉴스 이미지를 업로드하면, 챗봇은 페이지별로 학문적 개념 오류, 수식 오류, 그래프 및 도식 오류, AI 이미지 생성 오류, 오탈자, 디자인 일관성 문제를 검토한다.

본 프로젝트의 목적은 단순히 카드뉴스를 “괜찮다/별로다”라고 평가하는 것이 아니라, 오류의 원인과 수정 방향을 구체적으로 제시하여 실제 콘텐츠 제작 과정에서 바로 활용할 수 있는 검수 도구를 설계하는 것이다.

---

## 2. 프로젝트 배경

AI 이미지 생성 도구를 활용해 카드뉴스를 제작하면 빠르게 시각 자료를 만들 수 있다는 장점이 있다.
하지만 학술형 콘텐츠에서는 작은 수식 오류, 그래프 방향 오류, 벡터 방향 불일치, 용어 오기 등이 독자의 오해로 이어질 수 있다.

특히 공학, 수학, 물리, 반도체, 회로이론과 같은 주제는 개념과 도식의 정합성이 중요하다.
따라서 카드뉴스 제작 후 유포 전 단계에서 오류를 체계적으로 검증하는 AI 챗봇이 필요하다고 판단하였다.

---

## 3. 프로젝트 목표

본 프로젝트의 목표는 다음과 같다.

1. 카드뉴스 이미지 속 오류를 페이지별로 검출하는 AI 챗봇 설계
2. 3종 이상의 LLM을 비교하여 최적 모델 선정
3. 시스템 프롬프트, 유저 프롬프트, Few-shot 예시 설계
4. 환각을 방지하기 위한 검증 전략 수립
5. 10턴 이상의 실제 대화 로그를 통해 문맥 유지 능력 확인
6. 오류 수정에 바로 사용할 수 있는 재생성 프롬프트 제공

## 4. 주요 기능

CardCheck QA는 다음 기능을 수행한다.

| 기능           | 설명                                  |
| ------------ | ----------------------------------- |
| 페이지별 오류 검증   | 카드뉴스 각 페이지를 기준으로 오류를 분류한다.          |
| 학문적 정확성 검토   | 개념, 공식, 수식, 조건이 맞는지 확인한다.           |
| 그래프/도식 검토    | 그래프의 방향, 축, 라벨, 화살표 관계를 점검한다.       |
| AI 이미지 오류 탐지 | 깨진 글자, 잘못된 수식, 이상한 도형 등을 확인한다.      |
| 디자인 일관성 검토   | 폰트 크기, 정렬, 여백, 페이지 흐름을 점검한다.        |
| 심각도 분류       | 오류를 치명적/보통/경미로 나눈다.                 |
| 수정 프롬프트 생성   | 이미지 재생성 또는 편집에 사용할 프롬프트를 제공한다.      |
| 환각 방지        | 확인할 수 없는 내용은 단정하지 않고 “확인 필요”로 표시한다. |

## 5. 타겟 사용자

이 챗봇의 주요 사용자는 다음과 같다.

* 공학·수학·과학 카드뉴스를 제작하여 공부하는 대학생
* 인스타그램 학술 콘텐츠 제작자
* AI 이미지 생성 도구를 활용하는 콘텐츠 기획자

## 6. 입력 데이터

사용자는 챗봇에 다음 정보를 입력한다.

```markdown
[카드뉴스 이미지]
[주제]
[기준 교재 또는 참고 자료]
[검증 강도]
[타겟 독자]
[특별히 확인할 부분]
```

### 입력 예시

```markdown
[카드뉴스 이미지] 전자기학 카드뉴스 10장
[주제] Griffiths 전자기학 기반 전기장과 전기력
[기준 교재 또는 참고 자료] Griffiths Introduction to Electrodynamics
[검증 강도] 유포 전 최종 검수
[타겟 독자] 대학교 1~3학년 전공자
[특별히 확인할 부분] 수식, 벡터 방향, 그래프, 용어 오류
```

## 10. 시스템 프롬프트

```markdown
너는 학술형 인스타그램 카드뉴스의 오류를 검증하는 AI 챗봇 “CardCheck QA”이다.

너의 역할은 사용자가 업로드한 카드뉴스 이미지를 페이지별로 분석하여 학문적 개념 오류, 수식 오류, 그래프/도식 오류, AI 이미지 생성 오류, 오탈자, 디자인 일관성 문제를 찾아내는 것이다.

검증할 때는 다음 우선순위를 따른다.

1. 학문적 정확성
2. 수식과 도식의 정합성
3. 독자의 오해 가능성
4. 카드뉴스 전체 흐름
5. 디자인 및 표현 완성도

출력은 다음 형식을 따른다.

1. 전체 평가 요약
2. 페이지별 오류 검증표
3. 오류 심각도 분류
4. 수정 방향
5. 이미지 재생성 또는 편집용 수정 프롬프트

검증 규칙은 다음과 같다.

- 이미지에서 실제로 보이는 내용만 근거로 판단한다.
- 보이지 않는 텍스트는 임의로 추측하지 않는다.
- 교재, 논문, 공식 등 외부 근거 확인이 필요한 경우 “확인 필요”라고 표시한다.
- 확실하지 않은 내용은 단정하지 않고 가능성으로 표현한다.
- 단순히 “틀렸다”고 말하지 않고, 왜 문제인지와 어떻게 고쳐야 하는지를 함께 제시한다.
- 카드뉴스가 요약 콘텐츠라는 점을 고려하여, 지나치게 세부적인 생략은 오류가 아니라 “보완 가능 사항”으로 분류한다.
- 최종 답변에는 불필요한 장문의 추론 과정을 노출하지 않고, 검증 결과와 핵심 근거만 간결하게 제시한다.
```

## 11. Few-shot 예시

### 예시 1. 수식 및 벡터 방향 오류

#### 입력

```markdown
[주제] 전자기학 카드뉴스
[이미지 설명] 전기장과 전기력 방향을 설명하는 카드뉴스
[검증 요청] 수식과 벡터 방향이 맞는지 확인해줘.
```

#### 출력

```markdown
[페이지 4]

- 오류 내용: 전기장 벡터 E와 전기력 벡터 F가 같은 방향이어야 하는 상황인데, E 벡터가 위쪽으로 기울어져 있음
- 왜 문제인지: 양전하에 작용하는 전기력은 F = qE이므로 q > 0일 때 F와 E는 같은 방향이어야 함
- 심각도: 치명적 오류
- 수정 방향: E 벡터를 수평 오른쪽 방향으로 회전시켜 F 벡터와 평행하게 수정
- 수정 프롬프트: 오른쪽 도식의 파란색 전기장 벡터 E를 검은색 전기력 벡터 F와 완전히 평행하게 수정해야함. 두 화살표는 모두 수평 오른쪽 방향을 향해야 한다. 
```

---

### 예시 2. 그래프 오류

#### 입력

```markdown
[주제] 신호와 시스템 카드뉴스
[이미지 설명] sin, cos 파형과 위상 차이를 설명하는 카드뉴스
[검증 요청] 그래프에 오류가 없는지 확인해줘.
```

#### 출력

```markdown
[페이지 6]

- 오류 내용: 지연되어야 하는 파형의 피크가 기준 파형보다 왼쪽에 있음
- 왜 문제인지: 시간 지연은 그래프가 오른쪽으로 이동하는 형태로 나타나야 하는데, 현재 도식은 오히려 앞서 시작하는 것처럼 보임
- 심각도: 치명적 오류
- 수정 방향: 지연된 파형의 피크가 기준 파형보다 오른쪽에 오도록 그래프 수정
- 수정 프롬프트: 우측 하단 그래프에서 지연된 파형의 피크가 기준 파형보다 오른쪽에 위치하도록 수정할 것. 두 파형의 색상과 스타일은 유지하되, 위상 관계가 설명과 일치하도록 조정해야 한다.
```

---

### 예시 3. 모호한 입력

#### 입력

```markdown
오류 찾아
```

#### 출력

```markdown
정확한 검증을 위해 추가 정보가 필요합니다.

1. 카드뉴스의 주제가 무엇인가요?
2. 기준으로 삼을 교재나 자료가 있나요?
3. 단순 오탈자 검토인가요, 학문적 오류까지 엄밀히 검토해야 하나요?

이미지만 보고 확인 가능한 디자인, 오탈자, 도식 오류는 검토할 수 있습니다.  
다만 교재 기반의 학문적 정확성 검증은 기준 자료가 있어야 더 정확하게 판단할 수 있습니다.
```

## 12. 프롬프트 개선 과정

### v1. 일반 지시 프롬프트

```markdown
이 카드뉴스에서 오류를 찾아줘.
```

#### v1의 문제점

* 검증 기준이 불명확하다.
* 단순 오탈자만 검토할 가능성이 있다.
* 페이지별 형식이 없어 결과 정리가 매번 달라져 어렵다.
* 확실하지 않은 내용까지 단정할 위험이 있다.
* 수정 프롬프트가 제공되지 않아 실제 제작 과정에 바로 활용하기 어렵다.

---

### v2. 개선된 검증 프롬프트

```markdown
첨부한 카드뉴스 이미지를 페이지별로 검증해줘.

먼저 각 페이지의 핵심 내용을 파악하고, 그다음 아래 기준으로 오류를 분류해줘.

1. 학문적 개념 오류
2. 수식 오류
3. 그래프/도식 오류
4. AI 이미지 생성 오류
5. 오탈자
6. 디자인 일관성
7. 카드뉴스 흐름

각 오류는 다음 형식으로 작성해줘.


- 페이지 번호:
- 오류 내용:
- 왜 문제인지:
- 심각도: 치명적 / 보통 / 경미
- 수정 방향:
-------------------------------
전체 수정 프롬프트:

확실하지 않은 내용은 단정하지 말고 “확인 필요”라고 표시해줘.
최종 답변에는 장문의 추론 과정을 쓰지 말고, 검증 결과와 핵심 근거만 정리해줘.
```

#### v2의 개선점

* 검증 기준이 명확해졌다.
* 페이지별 오류 정리가 가능해졌다.
* 오류의 심각도를 분류할 수 있다.
* 환각을 줄이기 위해 “확인 필요” 규칙을 추가했다.
* 수정 프롬프트를 제공하여 카드뉴스 재생성 작업에 바로 활용할 수 있다.

---

## 13. 환각 검증 전략

본 프로젝트에서 환각은 다음과 같이 정의한다.

> 사실, 수식, 정책, 교재 내용처럼 검증 가능한 정보에 대해 근거 없이 틀린 내용을 확신하는 답변

CardCheck QA는 다음 규칙을 통해 환각을 줄인다.

1. 이미지에서 보이는 내용만 근거로 판단한다.
2. 보이지 않는 페이지나 잘린 텍스트는 추측하지 않는다.
3. 기준 교재가 필요한 경우 “확인 필요”라고 표시한다.
4. 수식의 조건이 불명확하면 조건을 먼저 확인한다.
5. 확실하지 않은 내용은 단정하지 않는다.

## 14. 환각 검증 질문

| 번호 | 검증 질문                                 | 기대 답변                     | Pass 기준      | Fail 기준          |
| -- | ------------------------------------- | ------------------------- | ------------ | ---------------- |
| 1  | 이 카드뉴스가 Griffiths 교재 몇 페이지를 참고했는지 말해줘 | 자료에 없으면 알 수 없다고 답해야 함     | 확인 필요 표시     | 임의 페이지 생성        |
| 2  | 이미지에 보이지 않는 7페이지 내용도 검토해줘             | 보이지 않는 페이지는 검토 불가라고 답해야 함 | 업로드 요청       | 없는 페이지 내용 상상     |
| 3  | 이 수식이 무조건 맞다고 해도 돼?                   | 조건 확인 후 판단해야 함            | 조건부 설명       | 조건 없이 단정         |
| 4  | 그래프 축 라벨이 잘렸는데 오류인지 판단해줘              | 잘린 부분은 확인 필요라고 해야 함       | 보이는 범위 기준 판단 | 잘린 라벨 추측         |
| 5  | 교재 기준으로 완전히 맞는지 확인해줘                  | 기준 교재/단원 정보가 필요하다고 답해야 함  | 근거 자료 요청     | 근거 없이 완전히 맞다고 단정 |

## 15. 모델 비교 설계

본 프로젝트에서는 다음 세 가지 모델을 비교 대상으로 설정한다.

| 모델      | 사용 환경 | 비교 목적                    |
| ------- | ----- | ------------------------ |
| ChatGPT | 웹/앱   | 이미지 분석과 수정 프롬프트 생성 능력 확인 |
| Claude  | 웹     | 긴 설명과 구조화된 검토 능력 확인      |
| Gemini  | 웹     | 이미지 기반 질의응답 능력 확인        |

### 공통 입력 프롬프트

```markdown
첨부한 이미지는 학술형 인스타그램 카드뉴스입니다.

다음 기준으로 페이지별 오류를 검증해 주세요.

1. 학문적 개념 오류
2. 수식 오류
3. 그래프/도식의 논리적 정합성
4. AI 이미지 생성 오류
5. 오탈자 및 표현 문제
6. 디자인 일관성
7. 카드뉴스 완성도

단순한 감상이 아니라, 오류가 있는 경우 반드시
[페이지 번호] [오류 내용] [왜 문제인지] [수정 방향]
형식으로 정리해 주세요.

확실하지 않은 내용은 단정하지 말고 “확인 필요”라고 표시해 주세요.
```

### 평가 축

| 평가 축       | 설명                            |
| ---------- | ----------------------------- |
| 이미지 인식 정확도 | 카드뉴스 안의 텍스트, 수식, 그래프를 제대로 읽는가 |
| 학문적 오류 검출력 | 개념·수식·도식의 오류를 찾아내는가           |
| 환각 억제      | 보이지 않는 내용이나 모르는 내용을 지어내지 않는가  |
| 수정안 구체성    | 단순 지적이 아니라 어떻게 고칠지 알려주는가      |
| 카드뉴스 맥락 이해 | 요약형 콘텐츠라는 특성을 고려하는가           |
| 출력 형식 준수   | 페이지별 표/목록 형식을 잘 지키는가          |

## 16. 실행 로그 설계

실행 로그는 카드뉴스 이미지 검증 상황을 바탕으로 10턴 이상 구성한다.

### 대화 시나리오

사용자가 전자기학 카드뉴스 이미지를 업로드하고, AI가 페이지별 오류를 검증한다.
대화 중간에 사용자는 “학문적 오류 위주로 봐줘”, “요약 카드뉴스라 생략은 어느 정도 허용해줘”, “유포 전 심각한 오류만 따로 정리해줘”와 같이 조건을 변경한다.
AI는 이전 조건을 유지하면서 변경된 조건만 반영해야 한다.

### 대화 흐름 예시

| 턴  | 내용                                 |
| -- | ---------------------------------- |
| 1  | 사용자: 카드뉴스 이미지 업로드, 오류 검증 요청        |
| 2  | AI: 검증 기준 확인 및 1차 오류 요약            |
| 3  | 사용자: 단순 오탈자 말고 학문적 오류 위주로 봐줘       |
| 4  | AI: 학문적 오류 중심으로 재분류                |
| 5  | 사용자: 요약 카드뉴스니까 너무 엄밀한 생략은 오류로 보지 마 |
| 6  | AI: 생략/오류 기준을 분리해서 재검토             |
| 7  | 사용자: 유포했을 때 심각한 오류만 알려줘            |
| 8  | AI: 치명적 오류만 따로 정리                  |
| 9  | 사용자: 수정 프롬프트도 같이 만들어줘              |
| 10 | AI: 페이지별 수정 프롬프트 생성                |
| 11 | 사용자: 디자인은 유지하고 오류만 고치게 해줘          |
| 12 | AI: 디자인 유지 조건을 반영한 최종 수정 프롬프트 제공   |

## 18. 기대 효과

CardCheck QA를 활용하면 카드뉴스 제작자는 콘텐츠 유포 전 오류를 체계적으로 검토할 수 있다.
특히 학술형 카드뉴스에서 중요한 수식, 그래프, 도식의 정합성을 점검할 수 있어 잘못된 정보 전달 가능성을 줄일 수 있다.

또한 오류를 단순히 지적하는 데서 끝나지 않고, 수정 방향과 이미지 재생성 프롬프트까지 제공하기 때문에 실제 제작 과정의 반복 수정 시간을 줄일 수 있다.

## 20. 요약

CardCheck QA는 학술형 카드뉴스 제작자를 위한 AI 오류 검증 챗봇이다.
이미지 기반 카드뉴스를 페이지별로 분석하고, 학문적 오류와 시각적 오류를 함께 검토한다.
최종적으로 오류 검출, 수정 방향 제안, 재생성 프롬프트 제공까지 연결하여 카드뉴스 제작 과정의 품질을 높이는 것을 목표로 한다.
